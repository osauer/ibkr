1. Executive summary

The deliverable is a Go-based ibkr CLI plus a small background daemon (ibkrd) that depends on a published shared library, github.com/osauer/ibkr-go, holding the TWS protocol implementation. Claude Code invokes the CLI via Bash; a bundled Skill documents available subcommands and the JSON output schemas. Permission rules pre-allow read-only invocations and a PreToolUse hook hard-blocks any trading subcommand until a deliberate v2 unlock.

Core architecture

  • TWS binary protocol via IB Gateway (port 4001 live / 7497 paper)
  • Daemon owns the gateway connection; CLI is a thin Unix-socket client
  • JSON-RPC over Unix socket, newline-delimited
  • JSON-file cache, no CGO, no external runtime deps

v1 scope

  • Account summary & positions (incl. options)
  • Stock and option quotes (snapshot & --watch)
  • Option chains (ATM ± N strikes)
  • Symbol-list quotes & preset scans
  • Built-in self-update against GitHub releases

v1 non-goals

  • Order placement / cancellation
  • Futures, FX, crypto, indices
  • Full IBKR scanner subscription DSL
  • Historical bar charting
  • Web/TUI dashboards

2. Goals & non-goals

Functional

Capabilityv1 behaviour
accountSnapshot of NLV, buying power, available funds, excess liquidity, margin, cash, currency.
positionsOpen positions only. Per-position: symbol, qty, avg cost, mark, market value, unrealized + realized P&L, day P&L %, account allocation %; option-aware columns (right/strike/expiry/Δ/IV/DTE) when applicable.
quote SYM[,SYM...]One-shot bid/ask/last/IV (when delivered) for stocks/ETFs and explicit option contracts. --watch streams.
chain SYM --expiryCalls + puts table, ATM ± --width strikes (default 5), Greeks where IBKR provides them.
scan <preset>Predefined IBKR scanner presets (top-movers, high-iv, etc.) plus user symbol lists in config.
statusDaemon health, gateway connection state, market-data type, profile, account.
self-updateFetch latest release from GitHub, verify signature, replace binaries atomically.

Non-functional

  • Latency. ≤ 100 ms p50 for cached commands; ≤ 1.5 s p50 for warm live snapshot. The TWS handshake amortises across the daemon's lifetime.
  • Honest data. No fabricated IV or Greeks. Every --json response carries data_type: live | delayed | frozen | delayed_frozen; absent fields stay null with status strings rather than synthesised.
  • Safety. Trading subcommand surface is unimplemented in v1. The order verb does not exist in the binary; permission rules and a PreToolUse hook refuse it independently.
  • Footprint. Pure-Go single module, builds for darwin/arm64 and linux/amd64.
  • Tested. Inherited regime test suite continues to pass against the extracted shared library; new daemon and CLI layers covered by unit and integration tests against a mock TWS server.

Explicit non-goals

  • Live or paper order entry. Confirmation, dry-run, kill-switch and audit-log surfaces deserve a dedicated design pass and are scheduled for v2.
  • TUI / web dashboards.
  • Multi-broker abstraction.
  • Closed-today positions in the default positions view (delivered separately as ibkr trades --today in v1.1).

3. Architecture

Two binaries, one shared library. The CLI never speaks the IBKR protocol directly; it talks JSON-RPC over a Unix socket to a daemon that holds the long-lived gateway connection, contract cache, and subscription state. 

   ┌──────────────┐         Unix socket            ┌──────────────────┐         TCP / TLS         ┌────────────────┐
   │   ibkr       │   JSON-RPC over $XDG_RUNTIME   │   ibkrd          │  ←————— port 4001 ——————→ │  IB Gateway    │
   │   (CLI)      │ ←——————————————————————————→ │   (daemon)       │       (or 7497 paper)     │  / TWS         │
   │   stateless  │   ~/.cache/ibkr/ibkrd.sock    │   stateful       │                           │                │
   └──────────────┘                                └────────┬─────────┘                           └────────────────┘
                                                            │

                                                │   shared Go module    │
                                                │   github.com/osauer/  │
                                                │     ibkr-go           │

Why a daemon

  • The TWS handshake costs 1–2 s and burns a client-ID slot; doing it per CLI invocation is wasteful and noisy in the gateway.
  • Streaming subscriptions need a long-lived socket.
  • A single owner of the client ID prevents the "another client is already connected" race.
  • The contract cache and the inactive-symbol set work best as in-memory state with a small persistence layer.

Why JSON-RPC over a Unix socket

  • Zero codegen, zero protobuf toolchain. Standard library only.
  • Trivially debuggable: socat - UNIX-CONNECT:.../ibkrd.sock + paste a JSON line.
  • Local-only IPC; no need for TLS, auth, or wire-compat across versions — daemon and CLI ship together.

4. The ibkr-go shared library

The TWS protocol implementation lives in its own repository (github.com/osauer/ibkr-go) from day one. The new CLI and the existing regime backtest/trading platform both depend on it via go.mod. Versioned independently with semver tags.

Origin and extraction

  1. Source: the package currently at regime/internal/components/ibkr. It implements connection pooling, contract resolution, market snapshots, market-data subscriptions, account summary, positions, order send (used by regime), and a mock TWS server for tests.
  2. Replace internal references to regime/internal/core types with package-native types (ibkr.Position, ibkr.Account, ibkr.Order, etc.).
  3. Replace regime/internal/logging.Component with a minimal Logger interface (slog-compatible), default no-op.
  4. Migrate regime's regime-specific concerns (e.g. orchestrator wiring, core.TickContext.ProcessTick) back into regime as a thin adapter that depends on the new module.
  5. During transition, both repos use a replace directive locally; once the public module is tagged, regime cuts over.
Public API line The shared module exposes only what both consumers genuinely need: connection lifecycle, contract resolution, snapshots, account summary, positions, subscriptions, and gated order send. Everything else stays unexported.

Public API

// github.com/osauer/ibkr-go

type Client interface {
    // Lifecycle
    Start(ctx context.Context) error
    Stop() error
    Status() Status
    // Reads
    Account(ctx context.Context) (*Account, error)
    Positions(ctx context.Context) ([]*Position, error)
    Snapshot(ctx context.Context, c Contract) (*Quote, error)
    Chain(ctx context.Context, req ChainRequest) (*Chain, error)
    // Streams
    Subscribe(ctx context.Context, c Contract) (<-chan Tick, CancelFunc, error)
    // Trading (compiled out in v1; gated by build tag)
    PlaceOrder(ctx context.Context, o Order) (*OrderAck, error)
    CancelOrder(ctx context.Context, id int) error

    Symbol     string
    SecType    string // STK | OPT
    Exchange   string // SMART
    Currency   string
    Expiry     time.Time  // zero for stocks

    Symbol         string
    Bid, Ask, Last *float64          // nil = not delivered
    IV             *float64

    AsOf           time.Time
}

Streaming throttle

quote --watch emits coalesced terminal output at --rate (default 250 ms). --rate 0 renders every tick. The internal IBKR subscription always runs at full gateway rate; throttling only affects terminal rendering, so subscribers reading the JSON stream over a pipe still see every frame.

6. Daemon & IPC

Lifecycle

  1. The CLI connects to the Unix socket at $XDG_RUNTIME_DIR/ibkr/ibkrd.sock (fallback ~/.cache/ibkr/ibkrd.sock).
  2. If the socket is missing or stale, the CLI autospawns ibkrd as a detached process and waits up to 5 s for socket readiness.
  3. ibkrd reads ~/.config/ibkr/config.toml, picks a profile (default paper), connects to the gateway, registers handlers, loads the cache files, and starts listening on the socket.
  4. Idle timer: the daemon shuts itself down after idle_timeout (default 30 min) of no client activity. ibkrd --foreground disables this for development.
  5. SIGTERM / SIGINT: drain in-flight requests, cancel subscriptions (cancelMktData), flush cache, close socket.

Wire format

Newline-delimited JSON. Each line is one complete JSON object. Streaming responses share an id and emit one frame per line, terminated by an end:true message.

// Request
{ "id": "r-7f2a", "method": "quote.snapshot",
  "params": { "symbol": "AAPL", "timeout_ms": 5000 } }

// Response (single)
{ "id": "r-7f2a", "ok": true,
  "result": { "symbol": "AAPL", "bid": 207.86, ... } }

// Streaming response
{ "id": "s-91bd", "ok": true, "stream": true,
  "frame": { "t": "2026-05-09T14:32:11Z", "bid": 207.86, "ask": 207.88 } }
{ "id": "s-91bd", "ok": true, "stream": true, "end": true }

// Error
{ "id": "r-7f2a", "ok": false,
  "error": { "code": "symbol_inactive", "message": "IBKR reported XYZ as inactive" } }

7. Cache layer

Stored data

  • Resolved contracts. Symbol → ConID, primaryExch, tradingClass, secType, currency, multiplier. Hundreds to a few thousand rows; ~100 bytes each.
  • Inactive symbol set. Symbol → reason + first_seen_at. Smaller still.

The cache is single-writer (the daemon owns it), small, and overwriteable. No joins, no range scans, no transactional consistency across files. v1 persists with plain JSON files, written via temp + os.Rename.

Engine choice

EngineVerdictReason
JSON files (stdlib)v1 defaultZero deps; atomic rename; jq-debuggable; fits the data shape.
bbolt (pure-Go embedded KV)upgrade pathReserved for if cache exceeds practical JSON limits or tick history is added. Same ContractCache interface, drop-in replacement.
SQLite (modernc or CGO)rejectedHeavyweight for the data; CGO version breaks the no-CGO constraint; pure-Go variant adds 6–10 MB to the binary for no gain.
In-memory onlyrejectedCold-start penalty: every contract re-resolved on each daemon restart.

Cache interface

type ContractCache interface {
    Get(symbol string) (Contract, bool)
    Put(c Contract)
    Delete(symbol string)
    All() map[string]Contract
    Flush(ctx context.Context) error   // idempotent; called on shutdown and on idle
}

3 · Settings snippet

SKILL.md (excerpt)

---
name: ibkr
description: Query Interactive Brokers via the local `ibkr` CLI. Use when the user asks
  about their IBKR account, positions, P&L, market quotes, option chains, or running a
  market scan. Read-only; never attempts to place orders.
allowed-tools: Bash(ibkr account*) Bash(ibkr positions*) Bash(ibkr quote*)
  Bash(ibkr chain*) Bash(ibkr scan*) Bash(ibkr status*)
---

## When to use

If the user asks about holdings, cash, buying power, P&L, a specific stock or ETF
quote, an option chain, or wants to scan the market, run the relevant `ibkr`
subcommand with `--json` and parse the output.

## Output discipline

- Always run with `--json` when parsing, then present results as a clean Markdown table.
- Always surface the `data_type` field (`live` / `delayed` / `frozen`). If it isn't
  `live`, mention it in the answer.
- Never claim an order was placed. The CLI cannot trade in v1.

## Commands

- `ibkr account --json` → account summary. Schema: see [schemas.md](schemas.md#account).
- `ibkr positions --json` → array of positions. Options have extra fields. Schema: see [schemas.md](schemas.md#positions).
- `ibkr quote AAPL --json` → single snapshot. Schema: see [schemas.md](schemas.md#quote).
- `ibkr chain AAPL --expiry YYYY-MM-DD --width 5 --json` → chain. Schema: see [schemas.md](schemas.md#chain).
- `ibkr scan top-movers --json` → list. Run `ibkr scan list` first if unsure.

## Errors

- `daemon_unavailable` → `ibkrd` could not start. Likely IB Gateway is not running.
- `symbol_inactive` → IBKR has flagged the symbol delisted/halted; do not substitute.
- `data_type: delayed` → quotes are delayed; warn the user before they act on them.

settings.json snippet

{
  "permissions": {
    "allow": [
      "Bash(ibkr account*)",
      "Bash(ibkr positions*)",
      "Bash(ibkr quote*)",
      "Bash(ibkr chain*)",
      "Bash(ibkr scan*)",
      "Bash(ibkr status*)"
    ],
    "deny": [
      "Bash(ibkr order*)",
      "Bash(ibkr trade*)",
      "Bash(ibkr cancel*)",
      "Bash(ibkr * --live*)"
    ]
  },
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "jq -e '.tool_input.command | test(\"\\\\bibkr\\\\s+(order|trade|cancel)\\\\b\")' >/dev/null && exit 2 || exit 0"
      }]
    }]
  }
}

The hook reads the tool-call JSON from stdin (Claude Code's contract), tests for trading verbs in the Bash command, and exits 2 to block. Exit 0 lets the call proceed. Requires jq on PATH.

Defence in depth Three independent layers refuse trading commands in v1: (a) the binary doesn't implement them; (b) the settings deny rule; (c) the PreToolUse hook. v2 lifts all three deliberately.

9. Self-update

ibkr self-update fetches the latest signed release from GitHub, verifies it, and atomically swaps the running binaries.

Behaviour

  1. ibkr self-update --check queries https://api.github.com/repos/osauer/ibkr/releases/latest and prints the available version vs. the current one. No mutation.
  2. ibkr self-update downloads the matching ibkr-{os}-{arch}.tar.gz asset plus its SHA256SUMS and SHA256SUMS.minisig sibling files.
  3. Verifies the minisign signature against a public key compiled into the binary, then verifies the tarball SHA-256 against the signed sums file.
  4. Extracts to a temp directory, then atomically replaces ibkr and ibkrd using os.Rename (with a backup-then-rename dance for in-use files on macOS / Linux).
  5. If ibkrd is running, sends SIGTERM and waits for graceful shutdown before swapping; the next CLI invocation autospawns the new version.
  6. --prerelease opts into pre-release tags. --yes skips the confirmation prompt for non-interactive use.

Version stamping

The build embeds version, commit SHA, and ISO build date via -ldflags: 

go build -ldflags "-X main.version=v1.0.0 -X main.commit=$(git rev-parse HEAD) -X main.date=$(date -u +%Y-%m-%dT%H:%M:%SZ)" ./cmd/ibkr

Safety

  • Refuses to run when the binary lives under a system-managed prefix (e.g. /usr/local/Cellar for Homebrew, /nix/store) — those paths are package-manager-owned.
  • Refuses prereleases unless --prerelease is set.
  • Verifies signature before extracting; bad signature aborts before any file is written into place.
  • Keeps the previous binary at ibkr.bak until the next successful run.

func main() {

}

// internal/daemon/handlers.go
func (s *Server) handleQuoteSnapshot(ctx context.Context, p QuoteSnapshotParams) (*ibkr.Quote, error) {
    contract, err := s.client.ResolveContract(ctx, p.Symbol)
    if err != nil { return nil, err }

    timeout := p.Timeout
    if timeout == 0 { timeout = 5 * time.Second }

    snap, err := s.client.Snapshot(ctx, contract)
    if err != nil { return nil, err }

    // Honest IV: do not interpolate, do not derive. If IBKR did not send tick 106,
    // IVStatus stays "unavailable" and the JSON field is null.
    return snap, nil
}

11.3 — Streaming subscription with proper cancellation

func (s *Server) handleQuoteSubscribe(
    ctx context.Context, p QuoteSubscribeParams, send func(Frame) error,
) error {
    contract, err := s.client.ResolveContract(ctx, p.Symbol)
    if err != nil { return err }

    ch, cancel, err := s.client.Subscribe(ctx, contract)
    if err != nil { return err }
    defer cancel()                  // always cancelMktData

    for {
        select {
        case <-ctx.Done():
            return ctx.Err()
        case tick, ok := <-ch:
            if !ok { return nil }       // connector closed stream
            if err := send(toFrame(tick)); err != nil {
                return err              // client gone → cancel runs via defer
            }
        }
    }
}

11.5 — JSON-file cache

// internal/cache/jsonfile.go
type JSONCache struct {
    path  string
    mu    sync.RWMutex
    data  map[string]Contract
    dirty bool
}

func (c *JSONCache) Get(symbol string) (Contract, bool) {
    c.mu.RLock(); defer c.mu.RUnlock()
    v, ok := c.data[symbol]
    return v, ok
}

func (c *JSONCache) Put(x Contract) {
    c.mu.Lock(); defer c.mu.Unlock()
    if existing, ok := c.data[x.Symbol]; ok && existing == x { return }
    c.data[x.Symbol] = x
    c.dirty = true
}

func (c *JSONCache) Flush(ctx context.Context) error {
    c.mu.RLock()
    if !c.dirty { c.mu.RUnlock(); return nil }
    snap := maps.Clone(c.data)
    c.mu.RUnlock()

    tmp := c.path + ".tmp"
    f, err := os.Create(tmp)
    if err != nil { return err }
    enc := json.NewEncoder(f)
    enc.SetIndent("", "  ")
    if err := enc.Encode(snap); err != nil {
        f.Close(); os.Remove(tmp); return err
    }
    if err := f.Close(); err != nil { return err }
    if err := os.Rename(tmp, c.path); err != nil { return err }

    c.mu.Lock(); c.dirty = false; c.mu.Unlock()
    return nil
}

11.6 — Self-update: signature verification & atomic swap

// internal/update/update.go
func (u *Updater) Apply(ctx context.Context, asset Asset) error {
    tarball, err := u.downloadVerified(ctx, asset)
    if err != nil { return err }
    defer os.Remove(tarball)

    // Verify the SHA256SUMS minisign signature against the embedded public key.
    if err := minisign.Verify(u.PublicKey, asset.SumsPath, asset.SigPath); err != nil {
        return fmt.Errorf("signature verification failed: %w", err)
    }
    if err := verifySHA256(tarball, asset.ExpectedSum); err != nil {
        return err
    }

    stagingDir, err := os.MkdirTemp("", "ibkr-update-")
    if err != nil { return err }
    defer os.RemoveAll(stagingDir)
    if err := extractTarGz(tarball, stagingDir); err != nil { return err }

    // Stop ibkrd if running so the binary is not in use during rename.
    _ = u.stopDaemon(ctx)

    for _, name := range []string{"ibkr", "ibkrd"} {
        target := filepath.Join(u.InstallDir, name)
        backup := target + ".bak"
        next   := filepath.Join(stagingDir, name)
        if err := os.Rename(target, backup); err != nil && !os.IsNotExist(err) {
            return err
        }
        if err := os.Rename(next, target); err != nil {
            _ = os.Rename(backup, target) // best-effort rollback
            return err
        }
        _ = os.Remove(backup)
    }
    return nil
}

14. Acceptance criteria & Definition of Done

Functional

  • ibkr account returns a populated NLV/BP/AvailFunds snapshot from a paper account in both text and --json.
  • ibkr positions shows open positions, with stocks and options separated and the documented column set; option Greeks are populated when IBKR delivers them and explicitly absent otherwise.
  • ibkr quote AAPL,MSFT,SPY returns three snapshots in < 2 s p50 against a warm daemon.
  • ibkr quote AAPL --watch --rate 250ms streams ≥ 5 distinct rendered windows during a market-open test window and exits cleanly on Ctrl-C; the gateway shows no orphan subscriptions afterwards.
  • ibkr quote AAPL --watch --rate 0 renders every tick.
  • ibkr chain AAPL --expiry <next-monthly> --width 5 returns 11 strikes (ATM ± 5) with calls and puts populated.
  • ibkr scan top-movers returns ≤ limit rows from the IBKR scanner.
  • ibkr status reports daemon uptime, connection state, profile, and data_type.
  • ibkr version prints version, commit, and build date.
  • ibkr self-update --check reports current and latest versions without side effects.
  • ibkr self-update applies a signed update and the resulting binary reports the new version; an invalid signature aborts the update with a non-zero exit code and no file change.

Non-functional

  • Cold-start ibkr quote AAPL ≤ 4 s end-to-end (handshake included).
  • Warm ibkr quote AAPL ≤ 1.5 s p50.
  • Daemon idle-shutdown after 30 min by default; configurable.
  • No fabrication: every --json response carries a non-empty data_type; IV is null with iv_status: "unavailable" when not delivered.
  • Trading verbs fail closed at three layers: binary stub, settings deny rule, PreToolUse hook.
  • go test ./... green in both ibkr-go and ibkr; CI runs on darwin and linux; binaries built without CGO.

Documentation

  • README.md: install, run, troubleshooting, gateway setup pointers, self-update workflow.
  • skill/SKILL.md + skill/schemas.md: every JSON field documented with example values.
  • CHANGELOG.md: v1.0.0 entry vs. baseline.
  • Inline godoc on every exported symbol of ibkr-go.

Integration verification

  • Manual smoke test against a paper account at IB Gateway 10.30+ documented in the release checklist.
  • Skill installed into a fresh ~/.claude/ demonstrates running ibkr account end-to-end without manual permission prompts.
  • Release artifact (binary + skill bundle) verified live on the GitHub releases page; checksums and signatures present.

16. Risks & mitigations

  • Shared-library extraction friction. The source package is woven with regime's logging and core types. Mitigation: extract on a feature branch; keep regime green throughout via a replace directive; cut over only after the new module is tagged.
  • Client-ID collisions. The daemon must use a different ID range from regime if both run on the same machine. Mitigation: defaults of 11 (paper) and 12 (live); regime uses 1–5.
  • Delayed-data confusion. Without entitlements a quote may be delayed. Mitigation: text formatter prints a yellow "DELAYED" badge; the Skill instructs Claude to surface this explicitly.
  • Daemon zombies. A crashed daemon could leave a stale socket. Mitigation: the CLI checks socket liveness with a one-shot status.health ping before reusing; reaps and respawns on timeout.
  • Self-update tampering. An attacker could serve a malicious release. Mitigation: minisign signature verification with a public key compiled into the binary; SHA-256 of each tarball asserted against the signed sums file before extraction.
  • Trading creep. The easiest mistake is to "just add a quick order command." Mitigation: trading is an entire build tag, an entire roadmap version, and an entire design pass — not a follow-up PR.
