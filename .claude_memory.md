## Working constitution — how Oliver wants work done

These were each learned from an explicit correction. Treat them as standing.

1. **Verify empirically before theorising.** Run the actual binary against
   live TWS (`ibkr chain SPY`, `ibkr status`, check the TWS UI) before
   hypothesising gateway/entitlement/off-hours causes. The bug is almost
   always in our code path. ("Did you not test with the binary?")
2. **Design doc + senior-reviewer critique before multi-step implementations.**
   A 30-minute reviewer-agent pass surfaces over-engineering and missing
   pieces cheaply. Don't land 3+ commits before a feedback loop.
3. **Measure before rewriting.** The Rust-port idea died when measurement
   showed Go CLI startup at 5.1 ms — within ~2 ms of the Rust floor, below
   human perception. Benchmarks beat language intuition; the cheap fix
   (lazy-load, `-ldflags="-s -w"`) won.
4. **No skip-workarounds for failing tests.** Skip-on-TWS is off-limits;
   find the root cause. (The May regression was a narrowed error-string
   match, not the scanner.)
5. **Honest UX beats eventual consistency.** "Come back later, cache is
   building" was rejected repeatedly. Anything regime/market-data must work
   whenever TWS is connected, expose explicit as-of timestamps and a
   "refreshing" status, and distinguish "asked and missed" from "didn't ask".
   No fake countdown ETAs, no `Status: ok` with silently-nil fields.
6. **Be honest about data limits; don't workaround-stack.** IBKR has no
   S&P breadth stats (S5FI/MMFI/etc. → "No security definition") — saying
   "IBKR can't do this" was the accepted answer, not bolting on FRED.
7. **Ruthless scope prioritisation.** When given a 5–10 item findings list,
   Oliver picks 2–3: "just put out the two worth doing." Lead with the
   highest-value items; don't pad.
8. **Interview, don't assume, on architecture.** For refactors (FSM in
   server.go, RPC envelope shape, Go modernization strategy) he wants to be
   educated and consulted on the choices, not handed a finished plan.
9. **Human voice in everything user-facing.** README, changelog, docs:
   plain English, humble, no jargon, senior-technical-editor taste pass.
10. **Upstream-first.** Server capabilities (rate limiting, middleware)
    belong in hyperserve, not re-implemented app-locally. Justify any
    hardening from the threat model or drop it.
11. **No decoy surfaces.** A flag the daemon hard-blocks anyway
    (`--bypass-preview`) is worse than no flag. Remove or document honestly.
12. **Solo-dev git flow.** Work directly on main, no feature branches.
    Parallel work → worktree-isolated agents. Pitfall: fresh worktrees base
    on origin/main, which lags local main — reset before agent work starts.
13. **No silent remote deploys.** Cloudflare relay changes are a separate
    gated go/no-go (compat matrix, rollback, staging). Never `wrangler deploy`
    as a side effect.
14. **Visual output needs a human-eye check.** CLI table/render changes
    that pass tests still shipped misaligned; run the command, screenshot,
    compare across all touched commands.

## Hard-won technical lessons

### IBKR API & market data
- **Timeout detection by error-string is fragile.** Commit d770f3d dropped
  `"context deadline exceeded"` from `isScannerTimeout` and broke live runs.
  Prefer `errors.Is(err, context.DeadlineExceeded)` over string lists.
- **Contract resolution is context-dependent.** `ibkr quote DIS` resolves;
  the breadth fetcher's bare `exchange=SMART` lookup misses the same names.
  Keep parity (primary-exchange hint, currency, sec-type) across all paths.
- **Historical-data pacing (60 req/10 min) collides with per-name timeouts.**
  Budget timeouts for the broker rate limit, not just network latency.
- **MinTick from contract details is the cross-venue minimum** (0.0001 on
  EUR names), not the lit grid — clamp stock ticks to a 0.01 floor.
- **Unhandled tick types are dropped silently** (e.g. 27/28 open-interest
  never populated `OpenInt`). When adding a field, trace the wire handler.
- **Wire tick ids ≠ generic-tick request ids** (2026-06-11): requesting
  generic tick 236 delivers wire tick 46 (difficulty, tickGeneric) + 89
  (share count, tickSize); generic 106 delivers wire tick 24 (underlying
  IV). Dispatching on the request ids left quote IV null and borrow
  inventory "unknown" for every symbol since the feature shipped. Never
  store tick 46's 0–3 float as shares — it reads as "Borrow scarce".
- **Account-scoped requests need a concrete account code** (2026-06-11):
  account-summary rows can carry the aggregate group "All"; sending it to
  reqAcctData/reqPnL gets TWS error 321 and a dead portfolio stream for
  the whole process (positions empty while quotes flow). Guard at the
  summary handler, the connector boundary, and postConnectSetup; a
  30s-throttled resubscribe watchdog in GetCachedPositions self-heals the
  residual TWS flake where a valid subscribe delivers nothing.
- **Boot races must not become persisted state** (2026-06-11): the
  proposal engine's startup refresh races the gateway connect — quick
  retries (30s→cadence backoff) clear the transient blocker, and
  error/unscoped shells are never persisted over the last-good snapshot.
  Same family: a gamma compute canceled by shutdown must abort, not
  degrade to a "successful" SPY-only result that gets persisted as the
  session-final read (it poisoned regime ranking for a full day).
- **Error 200 ("No security definition") must propagate** — otherwise the
  caller burns a full timeout waiting for data that will never come.
- **TWS API server ≥203 sends request errors as msg-204 system notices,
  not msgErrMsg frames** (2026-06-11): the entire msgID-4 recovery path
  (`handleIBKRError` fast-abort/historical-fail/slot-release/refresh,
  `handleErrorMessage` slot release) was dead code in production — 779
  system notices, zero msgErrMsg errors in one log sample. Consequences
  ran for months: historical fetches burned full timeouts then cancelled
  server-dead queries (recurring 366), RejectCh fast-abort never fired,
  and terminal 354s were re-requested every poll cycle (HGENQ trio:
  354+2129, then 300 on the cancel of the already-dead reqID).
  `recoverFromSystemNotice` now drives that recovery from the notice
  path; when adding error-driven behavior, wire it there, not (only) in
  the msgErrMsg handler.
- **Entitlement gaps get a 30-min in-memory absence window, never
  persistence** (2026-06-11): `Connector.mktDataAbsent` remembers a 354
  per subscription key (bare symbol or route key — record key must equal
  check key; alias-derived route keys never match and are how the old
  inactive marking missed HGENQ forever). Guards: skip option reqIDs
  (their key is the underlying), derivative aliases, unowned reqIDs, and
  impaired-farm windows (a bounce 354 says nothing about entitlement and
  farm bounces don't rebuild the connector). Suppressed subscribes fail
  fast with `MarketDataAbsenceError`; `handleQuoteSnapshot` converts it
  to the honest stale shell so the app's market_quotes source stays
  green. Wire-cancel of a server-rejected reqID is skipped via
  `rejectedReqID` match (never a sticky bool — refresh reassigns ReqID),
  slot release stays unconditional. The inactive-symbol marks got the
  same key contract later that day (2026-06-11 20:26 CEST):
  `subscriptionKeyForNotice` is the shared resolver for both memories,
  marking needs two definition errors inside `inactiveCandidateWindow`
  (10 min; stale candidates reset — one code-200 is routinely transient,
  and a single transient had suppressed an IDEALPRO FX-repair route),
  and `persistInactiveSymbol` refuses CASH/FX keys (`isCashRouteKey`) —
  the inverted direction of a pair legitimately has no definition, and a
  persisted entry could silently break positions FX-rate repair. The
  inactive store is a test-only hook; the daemon wires none.
- **rpc AssetType enums are not IBKR wire codes** (2026-06-11):
  `PositionView.SecType` carries "STOCK"/"OPTION" (see `positionSecType`);
  TWS rejects "STOCK" with error 321 'bG' "Please enter a valid security
  type". The proposal engine's min-tick contract-details fetch sent it
  verbatim — five held stock rows × every 15-min refresh, never cached
  because only successful fetches cache by conID, and
  `sendContractDetailsRequest` resolves by fields (conID goes out as 0
  even when known). Broker-bound params built from position rows must map
  through `positionWireSecType`; proposals/drafts already carry wire codes
  via `baseProposal`.
- **Subscription refcounting must be the only path.** Helpers that bypass
  SubManager caused concurrent-watcher drops; the missing `Observed` flag
  leaked rate-limiter slots until the 100-slot cap aborted gamma fan-out.
- **Scanner warmup is per-client-ID, not per-daemon.** Cold client IDs hang
  ~25 s on first scan; reuse client IDs (prod uses 15) in tests.
- **Off-hours quirks:** premarket BS-IV fallback waited 5 s/leg for ticks
  that never arrive (4× slower gamma); `data_type=frozen` is the right
  source for official prior-close in daily refreshes.

### Daemon / CLI / Go
- **Flag hoisting reorders args invisibly.** Subcommand routers must scan
  for the token, never check `args[0]` — leading-dash shortcuts otherwise
  silently run the default subcommand (exit 0, wrong behavior).
- **TOML unmarshal silently discards unknown keys.** Oliver's `[profiles]`
  section was dropped wholesale; reject unknown config keys loudly.
- **Empty enum values pass positive checks.** `rpc.IsLiveDataType("")`
  returned true and bypassed staleness gates in the purge path.
- **Sequential per-leg fetches under one global deadline compound** (5 s ×
  11 legs vs 55 s deadline = partial emergency close). Bounded parallel
  fan-out, like the 4-way positions fetch.
- **Daemon/CLI version skew shows up as context-deadline errors.** A stale
  daemon serving a new CLI is the first suspect; restart via
  `make install && ibkr restart`, never `pkill` for routine restarts.
- **Soft-TTL refresh + error-retry TTL can spawn duplicate computations**
  with reset progress under concurrent calls; single-flight the refresh.
- **Do work after acquiring the flock.** Eager cache-load/logging before
  `Server.Start` let autospawn-race losers spam the shared log.
- **SIGINT on `&` background jobs is SIG_IGN in non-interactive shells** —
  use `set -m` and confirm exit 130/137 when verifying interrupt teardown.

### Release engineering & trading safety
- `make release` is the only path (orchestrates spx-refresh → check/test →
  build → strict TWS release-smoke → tag → binaries → push → publish).
  A release without a live TWS smoke is a failed release.
- **Post-release, verify artifacts explicitly**: `gh release view`, registry
  curl, `git ls-remote --tags` — registry-publish strands silently.
- **Paper-smoke round-trip is a binding release gate** (since 2026-06-10),
  agent-runnable; not a runtime live blocker.
- Trading safety stack: paper open to agents; live writes hard-blocked
  daemon-side by origin; `ibkr settings set trading.freeze=true` as the
  human-only emergency brake; protection TIF policy is GTC (rollback =
  remove the tif key). Local builds are trading-capable by default;
  releases ship read-only + ibkr-trading variants — never add
  `GO_TAGS=trading` manually.
- **Live gate simplified 2026-06-11** (Oliver's call: order entry is
  latency-sensitive): config gate is mode="live" + gateway pins only.
  `allow_live`/`live_ack_*` and the typed `live/<account>` confirmation
  are gone end-to-end (daemon, RPC, CLI, app HTTP, SPA); leftover keys
  fail config load with a targeted error. SPA writes are single-click
  after preview; purge keeps its one typed confirm (bypass_preview, no
  token). The transitional tolerated `live_confirmation` DTO fields were
  dropped same-day — the phrase flow never reached a paired device.
- Changelog is tiered (What's new / KaC bullets / eng notes ≤15 lines),
  enforced by `make changelog-lint`; the GitHub release body auto-derives.

### Testing & verification
- **Test coverage ≠ production readiness.** Unit + daemon tests passed while
  the real binary failed install/lifecycle — hence the binary smoke gate.
- **Test teardown must reap daemons** (`t.Cleanup` + process-group kill);
  15 stale `ibkr daemon` processes once exhausted TWS API connections.
- **Match test context budgets to daemon deadlines** — a 15 s test ctx
  against a 35 s daemon path is a race window, not a test.
- `TestLifecycle_CLIDoesNotHangOnDeafDaemon` flakes under parallel `-race`
  load — verify in isolation, then re-run the full gate.
- **Dead code hides bugs.** Taste review found ~750–900 LOC of dead paths
  containing most HIGH-severity findings. Delete aggressively.
- **The browser smokes are manual gates, not release gates.** `app-smoke` /
  `app-lifecycle-smoke` are absent from the `check`/`test`/`release` chains;
  v1.9.0 shipped while they were red. They pin SPA DOM contracts by element
  id — when removing a `web/app` surface, grep `scripts/app-browser-smoke.mjs`
  and convert the assertion to a removal check (the 0574bd3 risk-plan removal
  stranded the mitigation-button assertion for two days, misread as a
  live-session bug). A fresh app instance's first canary poll can exceed 30 s
  against an off-hours live session; the asserted no-data contract is the
  "Waiting for canary snapshot" pending copy, not data presence.
